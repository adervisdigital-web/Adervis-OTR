Что берём в работу — выбери подход

Три области: вход, мобилка, ВКонтакте. Как расставить приоритеты?

💡 ВК уже частично готов: кнопка «📤 В VK» есть в чате, Edge Function развёрнута. Нужно: правильный UI + подтвердить webhook для входящих.
А

UX-спринт — всё сразу, но поверхностно

Новый дизайн входа + мобильная адаптация + улучшить кнопки ВК в чате.
Плюс: быстро, сразу видно улучшения везде.
Минус: ВК может не работать end-to-end до конца.

Б

ВК-first — сначала ВК работает полностью ✦ Рекомендую

Сделать ВК рабочим end-to-end: отправка из OTR → ВК ✓ / входящие из ВК → чат OTR ✓. Потом вход и мобилка.
Плюс: ежедневная работа станет реально удобнее уже сегодня.
Минус: дизайн входа пока остаётся как есть.

В

Качество-first — глубокая проработка всего

Новый экран входа + мобилка + ВК — всё сделать хорошо за несколько сессий. Полный редизайн.
Плюс: результат на уровне SaaS-продукта.
Минус: дольше, нужно несколько итераций.

Что входит в каждую область

🔐 Вход
  • Новый дизайн
  • Авто-сессия (уже есть)
  • Забыл пароль
  • Загрузочный экран
  • Стартовая подсказка
📱 Мобилка
  • Список лидов на весь экран
  • Чат как отдельный экран
  • Нижняя навигация
  • Удобный ввод с телефона
💬 ВКонтакте
  • Отправить из OTR → ВК
  • Входящие ВК → чат OTR
  • Авто-привязка peer_id
  • Статус «отправлено/ошибка»

Выбери подход и скажи в терминале что важнее всего
