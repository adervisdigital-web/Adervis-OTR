# Spec: Chat Interface Redesign

**Date:** 2026-06-12  
**Feature:** Диалог с клиентом — редизайн UX  
**Status:** Approved

---

## Контекст

Текущий интерфейс диалога (`#chat-view`) использует систему из двух табов («← Клиент ответил» / «✍️ Я написал») и двух колонок (история + поле ввода слева, скрипты справа). Это создаёт путаницу: непонятно в каком порядке действовать, кнопка «Записать» не описывает действие, скрипты далеко от поля ответа.

---

## Цель

Заменить текущий chat-view на линейный трёхшаговый поток, в котором порядок действий продажника очевиден без объяснений.

---

## Дизайн: Линейный поток «1 → 2 → 3»

### Общая структура

Одна колонка, сверху вниз:

```
┌─────────────────────────────────────┐
│  История диалога (прокручиваемая)   │
├─────────────────────────────────────┤
│  1. Клиент написал                  │
│     textarea + кнопка «Добавить»    │
├─────────────────────────────────────┤
│  2. Твой ответ                      │
│     Чипы скриптов + ✨ AI-вариант   │
│     [AI блок: 3 варианта с тоном]   │
│     textarea (редактируемый)        │
├─────────────────────────────────────┤
│  «📋 Скопировать»  «✓ Отправил»    │
├─────────────────────────────────────┤
│  📝 Заметки  ·  📅 Перезвонить     │
└─────────────────────────────────────┘
```

---

### Шаг 1 — «Клиент написал»

- **Заголовок:** фиолетовая метка `① КЛИЕНТ НАПИСАЛ` + подсказка «Вставь из VK / Inst / TG»
- **Поле:** `<textarea>` для вставки сообщения клиента. Очищается после добавления.
- **Кнопка:** `+ Добавить в историю` — добавляет сообщение в `lead.messages` с `fromClient: true`, обновляет ленту, очищает поле.

### Шаг 2 — «Твой ответ»

#### Чипы скриптов

- Ряд кнопок-чипов из существующих скриптов (`adervis_custom_scripts_v4`).
- Показываются **релевантные** скрипты: фильтруются по платформе лида (VK / Inst / TG) — первые 4 штуки.
- При клике — текст скрипта (с подставленными CTA-плейсхолдерами) вставляется в textarea ответа.
- Последний чип: `✨ AI-вариант` (зелёный).

#### AI-блок (состояние: свёрнут по умолчанию)

- Появляется после клика на `✨ AI-вариант`.
- Вызывает Gemini API с промптом: последнее сообщение клиента + контекст лида (bizType, platform). Промпт явно просит вернуть ровно 3 варианта в формате `1. [МЯГКИЙ] текст`, `2. [ДЕЛОВОЙ] текст`, `3. [ВОПРОСОМ] текст` — разделённых переносом строки.
- Ответ парсится регуляркой по маркерам `[МЯГКИЙ]`, `[ДЕЛОВОЙ]`, `[ВОПРОСОМ]` на 3 варианта с тонами: **Мягкий / Деловой / Вопросом**. Если парсинг не удался — показывается весь текст как один вариант.
- Каждый вариант — кликабельная карточка. Клик → текст вставляется в textarea ответа, AI-блок сворачивается.
- Если ключ Gemini не задан → кнопка ведёт в настройки.

#### Textarea ответа

- Редактируемое поле для финального текста ответа.
- Placeholder: «Нажми скрипт выше — или напиши свой вариант...»
- При фокусе — подсвечивается фиолетовым бордером.

### Шаг 3 — Отправка

- **`📋 Скопировать`** — копирует текст из textarea в буфер. После копирования кнопка на 2с показывает «Скопировано ✓».
- **`✓ Отправил`** — добавляет текст в `lead.messages` с `fromClient: false`, очищает textarea ответа, прокручивает ленту вниз.

### Нижняя полоса (без изменений)

Заметки + напоминание — остаются как есть.

---

## Что убирается

| Убирается | Причина |
|---|---|
| Табы «← Клиент» / «✍️ Менеджер» | Заменяются двумя раздельными шагами с явными действиями |
| Правая колонка `#scriptPanel` | Скрипты переезжают в чипы шага 2 |
| Кнопка «Записать» | Заменяется «Добавить в историю» и «✓ Отправил» |
| Подсказка «Ctrl+Enter — записать» | Заменяется явной кнопкой |
| Переменная `chatInputTab` | Больше не нужна — два отдельных textarea |

---

## Что остаётся без изменений

- Структура данных `lead.messages[]` — не меняется
- Функции `addMessageToLead`, `renderMessagesFeed`, `renderSingleMessage` — не меняются
- Редактирование и удаление сообщений в ленте — не меняются
- `saveNotes`, `saveReminder` — не меняются
- Gemini API (`generateAiReply`) — переиспользуется, добавляется парсинг на 3 варианта

## CSS — новые классы

```
.chat-step          — блок шага (padding, border-bottom)
.chat-step-header   — заголовок шага (номер + label)
.step-num           — кружок с цифрой
.script-chips       — flex-ряд чипов
.chip               — кнопка-чип (скрипт)
.chip-ai            — зелёный чип AI
.ai-variants        — блок с тремя AI-вариантами (скрытый по умолчанию)
.ai-variant-card    — одна карточка варианта
.ai-tone-label      — метка тона (МЯГКИЙ / ДЕЛОВОЙ / ВОПРОСОМ)
.chat-send-bar      — нижняя панель с кнопками «Скопировать» и «Отправил»
```

## Изменения в JS

| Функция | Изменение |
|---|---|
| `setChatInputTab` | Удаляется |
| `submitChatInput` | Разделяется на `submitClientMsg(leadId)` и `submitManagerMsg(leadId)` |
| `renderScriptChips(lead)` | Новая: рендерит чипы под платформу лида |
| `handleChipClick(scriptText)` | Новая: вставляет текст в `#replyTextarea` |
| `toggleAiVariants(leadId)` | Новая: вызывает Gemini, показывает/скрывает блок |
| `selectAiVariant(text)` | Новая: вставляет выбранный вариант в `#replyTextarea` |
| `copyReply()` | Новая: копирует `#replyTextarea` в буфер |
| `openChatView(leadId)` | Обновляется: рендерит новую структуру шагов |

## Доступность (WCAG 2.2 AA)

- Все интерактивные элементы — `<button>`, не `<div>`
- `aria-label` на чипах: «Вставить скрипт: {название}»
- AI-блок: `aria-live="polite"` при появлении вариантов
- Кнопка «✓ Отправил» после клика анонсирует «Сообщение записано» через `aria-live`
- Каждый шаг имеет `<h3>` для навигации скринридером

## Состояния UI

| Состояние | Описание |
|---|---|
| Пустой диалог | Оба textarea пустые, кнопки «Добавить» и «Отправил» задизаблены |
| После клика AI | AI-блок раскрывается под чипами, кнопка AI — спиннер пока грузится |
| После клика варианта | AI-блок сворачивается, textarea заполняется |
| После «Отправил» | Textarea очищается, лента обновляется, появляется анимация ✓ |
