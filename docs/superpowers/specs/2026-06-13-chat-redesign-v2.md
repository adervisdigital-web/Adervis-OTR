# UX Improvements v2 — ADERVIS OTR

**Дата:** 2026-06-13
**Файл:** `Adervis LidGen.html` (single-file vanilla JS)

---

## Проблема

Текущий чат-экран непонятен новым пользователям:
- Карточки шаблонов не имеют явного действия («как копировать?»)
- Кнопка «Новый → Ледокол» непонятна (только вперёд, нельзя редактировать статус свободно)
- Два черновика и четыре кнопки (Копировать / Отправлено / Скопировать / Записать) создают путаницу

---

## Дизайн

### 1. Шапка чата

```
← Кофейня «Уют» · ВК          Статус: [В диалоге ▾]
```

- Кнопка «Назад» (←) — возврат к таблице лидов
- Название лида + платформа-бейдж
- **Дропдаун статуса** справа: Новый / Ледокол / В диалоге / Успех / Отказ — любое значение в любом направлении
- Кнопка «Новый → Ледокол» и аналоги **удаляются**

### 2. Левая колонка — лента сообщений

Без изменений. Пузырьки: серые (клиент, слева) / фиолетовые (менеджер, справа).

**Ввод внизу** — два таба + textarea (остаётся как есть):
- `← Клиент ответил` — записывает как сообщение клиента
- `✍️ Я написал` — записывает как сообщение менеджера
- Кнопка «Записать» + Ctrl+Enter

### 3. Правая колонка — шаблоны (полный рефакторинг)

**Фильтр-чипы по этапу** (горизонтально):
```
[Ледокол] [Ответ на возражение] [Напоминание]
```
Чипы берут названия из существующих этапов скриптов. Активный чип подсвечен.

**Карточки шаблонов:**
```
┌──────────────────────────────────────────┐
│ Привет! Мы снимаем короткие видео для    │
│ кафе и ресторанов. Хотите посмотреть     │
│ примеры работ?                           │
│                                    [⿻]  │
└──────────────────────────────────────────┘
```

- `[⿻]` — SVG-иконка copy (два прямоугольника), 14×14px, цвет `var(--muted)`, при hover → `var(--text)`
- Клик по иконке: **копирует текст в буфер обмена** + **записывает в историю чата** как сообщение менеджера + показывает тост «Скопировано ✓» на 2 сек
- Клик по тексту карточки — выделяет/раскрывает для ручного копирования (нет авто-действия)

**AI-кнопка:**
```
[✨ Сгенерировать ответ]
```
- Вызывает Gemini API с последним сообщением клиента
- Результат появляется как временная карточка с той же SVG-иконкой copy
- Временная карточка исчезает при смене лида или закрытии чата

**Черновик (textarea) — удаляется** вместе с кнопками «Копировать» и «Отправлено».

### 4. SVG-иконка copy

Используется инлайн SVG (не эмодзи):
```html
<svg width="14" height="14" viewBox="0 0 24 24" fill="none" stroke="currentColor" stroke-width="2">
  <rect x="9" y="9" width="13" height="13" rx="2"/>
  <path d="M5 15H4a2 2 0 0 1-2-2V4a2 2 0 0 1 2-2h9a2 2 0 0 1 2 2v1"/>
</svg>
```

## Поведение дропдауна статуса

- Открывается по клику на `[В диалоге ▾]`
- Варианты: Новый / Ледокол / В диалоге / Успех / Отказ
- При выборе — сохраняется через `setStatus(leadId, n)` + обновляет бейдж в шапке
- Закрывается по клику вне или Escape

## Глобальные CSS-тултипы

### Проблема
Иконки-кнопки (⚙️ ℹ️, copy-иконка, ✕ Отказ) не объясняют своё назначение без подписи.

### Решение
Единый CSS-компонент `[data-tooltip]` — добавляется к любому элементу через атрибут:

```html
<button data-tooltip="Настройки шаблонов">⚙️</button>
```

**Внешний вид:**
```
        ┌───────────────────┐
        │ Настройки шаблонов│  ← тёмный фон #1a1b1e,
        └─────────┬─────────┘    белый текст, 8px radius
                  ↓
              [⚙️]
```

**CSS-реализация** (псевдоэлементы, без JS):
```css
[data-tooltip] { position: relative; }
[data-tooltip]::after {
  content: attr(data-tooltip);
  position: absolute; bottom: calc(100% + 6px); left: 50%;
  transform: translateX(-50%);
  background: #1a1b1e; color: var(--text); font-size: 11px;
  padding: 4px 8px; border-radius: 6px; white-space: nowrap;
  border: 1px solid var(--line); pointer-events: none;
  opacity: 0; transition: opacity .15s;
  font-family: var(--font-ui);
}
[data-tooltip]:hover::after { opacity: 1; }
```

**Где добавить `data-tooltip`:**

| Элемент | Текст тултипа |
|---------|---------------|
| `⚙️` кнопка в хедере | «Редактор скриптов» |
| `ℹ️` кнопка в хедере | «Справка» |
| `📁 Данные ▾` | «Экспорт и бэкап» |
| Copy-иконка на карточке | «Скопировать и записать в историю» |
| `✕` быстрый отказ в таблице | «Отметить как Отказ / Игнор» |
| `×` удалить лид | «Удалить лид (без возврата)» |
| Счётчик попыток `📤 3` | «Количество попыток связаться» |
| Бейдж платформы (VK / Inst) | «ВКонтакте» / «Instagram» / «Telegram» |

## Сегмент в bulk import

### Проблема
При загрузке списка лидов через «+ Загрузить лиды» сегмент (bizType) не задаётся — все лиды импортируются без категории, надо вручную ставить каждому.

### Решение
Добавить выбор сегмента в верхнюю часть модального окна bulk import. Сегмент применяется ко всем импортируемым лидам.

**Дизайн модального окна:**
```
┌─────────────────────────────────────────┐
│ Загрузить лиды                      [×] │
├─────────────────────────────────────────┤
│ Сегмент бизнеса:                        │
│ [Не указан ▾]                           │
│  • Общепит (кафе, рестораны, бары)      │
│  • Красота и уход                       │
│  • Фитнес и спорт                       │
│  • Розница                              │
│  • Услуги                               │
│  • Другое                               │
├─────────────────────────────────────────┤
│ Список ссылок (каждая с новой строки):  │
│ ┌─────────────────────────────────────┐ │
│ │ https://vk.com/cafe_uut             │ │
│ └─────────────────────────────────────┘ │
│                           [Просмотреть] │
└─────────────────────────────────────────┘
```

**Поведение:**
- Дропдаун использует те же значения что и фильтр по сегментам в таблице
- По умолчанию — «Не указан» (пустое значение, поведение как раньше)
- При импорте: выбранный сегмент записывается в `lead.bizType` для каждого нового лида
- Дубликаты (уже существующие по URL): сегмент **не перезаписывается**
- В preview-экране (после «Просмотреть») показывается выбранный сегмент рядом с каждым лидом
