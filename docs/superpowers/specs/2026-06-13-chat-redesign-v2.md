# Chat View Redesign v2 — ADERVIS OTR

**Дата:** 2026-06-13
**Файл:** `Adervis LidGen.html` (single-file vanilla JS)

---

## Проблема

Текущий чат-экран непонятен новым пользователям:
- Карточки шаблонов не имеют явного действия («как копировать?»)
- Кнопка «Новый → Ледокол» непонятна (только вперёд, нельзя редактировать статус свободно)
- Два черновика и четыре кнопки (Копировать / Отправлено / Скопировать / Записать) создают путаницу

---

## Дизайн

### 1. Шапка чата

```
← Кофейня «Уют» · ВК          Статус: [В диалоге ▾]
```

- Кнопка «Назад» (←) — возврат к таблице лидов
- Название лида + платформа-бейдж
- **Дропдаун статуса** справа: Новый / Ледокол / В диалоге / Успех / Отказ — любое значение в любом направлении
- Кнопка «Новый → Ледокол» и аналоги **удаляются**

### 2. Левая колонка — лента сообщений

Без изменений. Пузырьки: серые (клиент, слева) / фиолетовые (менеджер, справа).

**Ввод внизу** — два таба + textarea (остаётся как есть):
- `← Клиент ответил` — записывает как сообщение клиента
- `✍️ Я написал` — записывает как сообщение менеджера
- Кнопка «Записать» + Ctrl+Enter

### 3. Правая колонка — шаблоны (полный рефакторинг)

**Фильтр-чипы по этапу** (горизонтально):
```
[Ледокол] [Ответ на возражение] [Напоминание]
```
Чипы берут названия из существующих этапов скриптов. Активный чип подсвечен.

**Карточки шаблонов:**
```
┌──────────────────────────────────────────┐
│ Привет! Мы снимаем короткие видео для    │
│ кафе и ресторанов. Хотите посмотреть     │
│ примеры работ?                           │
│                                    [⿻]  │
└──────────────────────────────────────────┘
```

- `[⿻]` — SVG-иконка copy (два прямоугольника), 14×14px, цвет `var(--muted)`, при hover → `var(--text)`
- Клик по иконке: **копирует текст в буфер обмена** + **записывает в историю чата** как сообщение менеджера + показывает тост «Скопировано ✓» на 2 сек
- Клик по тексту карточки — выделяет/раскрывает для ручного копирования (нет авто-действия)

**AI-кнопка:**
```
[✨ Сгенерировать ответ]
```
- Вызывает Gemini API с последним сообщением клиента
- Результат появляется как временная карточка с той же SVG-иконкой copy
- Временная карточка исчезает при смене лида или закрытии чата

**Черновик (textarea) — удаляется** вместе с кнопками «Копировать» и «Отправлено».

### 4. SVG-иконка copy

Используется инлайн SVG (не эмодзи):
```html
<svg width="14" height="14" viewBox="0 0 24 24" fill="none" stroke="currentColor" stroke-width="2">
  <rect x="9" y="9" width="13" height="13" rx="2"/>

</svg>
```

## Удаляемые элементы

| Элемент | Причина |
|---------|---------|
| Кнопка «Новый → Ледокол» (и аналоги) | Заменена дропдауном статуса в шапке |
| `#chatDraftArea` textarea | Не нужна — копирование прямо с карточки |
| Кнопка «📋 Копировать» | Заменена иконкой на карточке |
| Кнопка «✓ Отправлено» | Копирование автоматически записывает в историю |
| `window._chatDraft` storage | Черновик удаляется |

---

## Изменения в данных

Нет. `lead.messages[]` и `lead.status` — без изменений.

---

## Поведение дропдауна статуса

- Открывается по клику на `[В диалоге ▾]`
- Варианты: Новый / Ледокол / В диалоге / Успех / Отказ
- При выборе — сохраняется через `setStatus(leadId, n)` + обновляет бейдж в шапке
- Закрывается по клику вне или Escape

---

## Тост-уведомление

При копировании с карточки — небольшое уведомление внизу экрана:
```
✓ Скопировано и записано
```
Появляется 2 сек, исчезает. Реализуется через `aria-live="polite"` для доступности.
